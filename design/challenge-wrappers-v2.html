How to Build This

A. CSS + Inline SVG

Everything you see above. Gradients, pseudo-elements, inline SVG patterns, CSS animations. Zero external assets. Ship as a theme JSON + a small StyleSheet.

Pro: Zero load time, tiny bundle, works offline, AI can generate themes as pure code.

Con: Limited to what CSS can draw. No photorealistic snow or complex illustrations.

RECOMMENDED START

B. AI-Gen Card BGs

Generate a 200x200 dark background image per theme via DALL-E/Midjourney. Snow scene, heart bokeh, spiderweb macro. Icon renders on top in white.

Pro: Photorealistic, unlimited creativity, unique per challenge.

Con: ~15KB per image x 4 cards x N themes. Need to cache/preload. Quality control on AI output.

FUTURE UPGRADE

C. Lottie Overlays

Animated snowfall, floating hearts, flickering flames as lightweight Lottie JSON inside each card. You already have lottie-react-native.

Pro: Smooth 60fps animations, designer-friendly (After Effects export), small files.

Con: Need to source/create each animation. 4 Lotties per card row = perf consideration.

STRETCH

Recommended approach: Start with A (CSS-only) — it covers 80% of the visual impact with zero asset overhead. The snow, hearts, webs, and waves above are all pure CSS.

Later, upgrade specific high-value themes to B (AI background) or C (Lottie) when you want that extra 20%. The card component just needs a theme prop — the rendering approach is an implementation detail behind it.
